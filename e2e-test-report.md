## 3. Bug 修复记录

### 3.1 Bug：点单页商品列表始终为空

**现象**：
- 首页 → 酒吧详情 → 点单，商品列表和分类均为空
- 但直接调用云函数 `getProductList` 和 `getCategories` 能正常返回数据（10 个商品、4 个分类）
- 真机调试正常，仅模拟器复现

**根因**：
`src/pages/order/index.vue` 仅在 `onMounted` 中加载数据。由于「点单」是 tabBar 页面：
1. 用户首次进入「点单」页时 `currentBar` 为空，`onMounted` 直接 return
2. tabBar 页面被 Vue 缓存
3. 用户从「酒吧详情」通过 `uni.switchTab` 跳转回「点单」页时，**`onMounted` 不会再次触发**
4. 导致即使 `currentBar` 已有值，分类和商品也永远不会加载

**修复方案**：`src/pages/order/index.vue`

```js
// 增加 onShow 钩子，在 tabBar 页面重新显示时补充加载数据
let onShowHook = null

const initData = async () => {
  if (!currentBar.value) return
  await fetchCategories()
  await fetchProducts()
}

onMounted(() => {
  initData()
  onShowHook = () => { initData() }
  const pages = getCurrentPages()
  const thisPage = pages[pages.length - 1]
  if (thisPage) {
    const originalOnShow = thisPage.onShow
    thisPage.onShow = function () {
      if (originalOnShow) originalOnShow.call(this)
      if (onShowHook) onShowHook()
    }
  }
})

onUnmounted(() => {
  onShowHook = null
})
```

**修复验证**：修复后分类列表和商品卡片正常渲染，切换分类、加入购物车均正常。
